SQL Injection vulnerability in user search endpoint

security Rule: S3649 2025-12-29 10:30 In fase da 2h 15m

Descrizione

L'endpoint /api/users/search utilizza concatenazione di stringhe per costruire query SQL, permettendo attacchi di SQL injection.

Un attaccante potrebbe iniettare codice SQL malevolo attraverso il parametro query per accedere o modificare dati sensibili.

Posizione

src/api/routes/users.py : Line 45-52

Codice Problematico 

query = f"SELECT * FROM users WHERE name LIKE '%{search_term}%'"
cursor.execute(query)  # VULNERABLE!

Fix Suggerito 

query = "SELECT * FROM users WHERE name LIKE %s"
cursor.execute(query, (f"%{search_term}%",))  # Parameterized!
