FUNC-MED-004

MEDIUM open

Missing input validation in user registration endpoint

security Rule: B608 Creata: 2024-01-15 14:32

Descrizione

The user registration endpoint at /api/v1/users/register does not properly validate input data before processing. This could allow malicious users to inject invalid data or potentially exploit SQL injection vulnerabilities.

The endpoint accepts user input directly without sanitization, which violates OWASP security guidelines for input validation.

Posizione nel codice

src/api/routes/users.py : Line 45-52

Codice Problematico 

@router.post("/register")
async def register_user(request: Request):
    data = await request.json()
    # ISSUE: No validation here!
    username = data["username"]
    email = data["email"]
    password = data["password"]

    # Direct DB insert without sanitization
    await db.execute(f"INSERT INTO users VALUES ('{username}', '{email}', '{password}')")

Fix Suggerito 

from pydantic import BaseModel, EmailStr, validator

class UserRegisterRequest(BaseModel):
    username: str
    email: EmailStr
    password: str

    @validator('username')
    def validate_username(cls, v):
        if len(v) < 3 or len(v) > 50:
            raise ValueError('Username must be 3-50 characters')
        return v

@router.post("/register")
async def register_user(data: UserRegisterRequest):
    # Pydantic handles validation automatically
    await db.execute(
        "INSERT INTO users VALUES (:username, :email, :password)",
        {"username": data.username, "email": data.email, "password": hash_password(data.password)}
    )
